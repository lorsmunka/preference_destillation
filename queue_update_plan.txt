QUEUE-BASED PIPELINE UPGRADE PLAN (v4)
=======================================

GOAL: Multi-domain distillation pipeline. Queue-based config for data generation,
      distillation data generation (Gemma logits), and training. Two domains:
      reddit comment sentiment classification + math word problem solving.

WORKFLOW: Claude makes changes, user reviews, user stages+commits.
Claude does NOT stage or commit. User controls git entirely.


PIPELINE OVERVIEW (3 stages)
------------------------------
1. TEXT GENERATION     -> Domain-specific input creation (Reddit extraction, math problem generation)
2. DISTILLATION GEN   -> Gemma processes inputs, autoregressively generates with logit capture per step
3. TRAINING           -> Student transformer learns from teacher logits

PHASE 1: TEXT GENERATION RESTRUCTURE + MATH PROBLEM GENERATOR [COMPLETE]
========================================================================

Directory structure created:
  text_generation/
    reddit_comment_sentiment/
      get_reddit_comments.py    (moved+renamed from ./get_sentences.py)
      reddit_comments.jsonl     (moved+renamed from ./sentences/sentences.jsonl)
    math_word_problem/
      word_problem_variables.py (names, semantic item categories, singularize)
      operations.py             (operation types, number generation, scaffolds, distribution)
      templates.py              (260+ templates across 13 operation types)
      generate_problems.py      (generates 500k unique problems -> math_word_problems.jsonl)

Key decisions made:
  - JSONL format identical for both domains: {"text": "..."}
  - No answer field needed — teacher logits ARE the ground truth
  - Semantic item categories (BAKEABLES, CONSUMABLES, ANIMALS, MANUFACTURED, etc.)
    prevent nonsense like "bakes 80 cats" or "factory makes strawberries"
  - ANIMALS excluded from ALL_ITEMS, only used via {animals} in farm/zoo templates
  - {container} = singular, {containers} = plural for grammar correctness
  - fix_singular_grammar uses regex word boundaries to avoid substring bugs
  - Division: divisor capped at range_high//2, min quotient=2 to avoid trivial results
  - Gemma 4B limitation: max 3 input variables, 1 operation before comparison

13 operation types with scaffolds:
  add, sub, mul, div (2 vars)
  mul_add, mul_sub, add_mul, sub_mul, add_sub, div_add (3 vars, 2 steps)
  cmp, cmp_sub, cmp_mul (comparison, True/False output)

Number distribution (weighted random):
  20% single digits (1-9, messy)
  15% tens round (10,20,...90)
  10% tens messy (13,47,68,...)
  15% hundreds round (100,200,...900)
  10% hundreds messy (137,299,483,...)
  15% comparison tens round
  15% comparison tens messy

For hundreds+ ranges, only add/sub/div/add_sub/div_add (no mul — too large results).

Master prompt for math (sent to Gemma, student NEVER sees this):
  Follow this example exactly:

  Problem: "Lisa has 3 bags with 5 apples each. She eats 2 apples. How many are left?"
  A=3
  B=5
  C=2
  D=A*B=15
  E=D-C=13
  Solution: 13

  Problem: "Tom has 15 stickers. He gives away 7. Sarah has 12 stickers. Does Tom have more stickers than Sarah?"
  A=15
  B=7
  C=12
  D=A-B=8
  E=D>C=False
  Solution: False

  {generated problem text with scaffold}

PHASE 2: DISTILLATION DATA GENERATION (queue system for data_generation/)
========================================================================

OVERVIEW:
  Current state: data_generation/ is hardcoded for reddit sentiment only.
  - main.py loops through sentences, calls model_handler for each
  - model_handler.py loads Gemma, creates sentiment vocabulary, generates autoregressively
  - sentence_handler.py loads reddit_comments.jsonl
  - saving_handler.py writes batches to BATCHES_DIR (hardcoded "./batches12b")
  - shared/utilities.py has sentiment-specific vocabulary (EXAMPLE_JSON_RESPONSES, etc.)
  - shared/config.py has hardcoded MODEL_NAME, BATCHES_DIR, MAX_TRAINING_EXAMPLES

  Target state: queue-driven, domain-aware pipeline.
  - data-generation-queue.json defines what to process
  - Each queue element: domain, model_name, max_examples, batch_size, description
  - Domain determines: input file, prompt construction, output vocabulary, stop condition
  - Output: batches/{domain}/{sanitized_model_name}/batch_N.jsonl + info.json

FILES THAT NEED CHANGES:
  data_generation/main.py           -> queue loop, domain dispatch
  data_generation/model_handler.py  -> domain-aware vocab + prompt, configurable model
  data_generation/sentence_handler.py -> renamed to input_handler.py, loads by domain
  data_generation/saving_handler.py -> domain/model output paths
  shared/config.py                  -> remove hardcoded MODEL_NAME, BATCHES_DIR
  shared/utilities.py               -> domain-dispatched vocabulary builder
  NEW: data-generation-queue.json   -> queue config file
  NEW: shared/math_vocabulary.py    -> math domain vocabulary (or section in utilities.py)

      "domain": "reddit_comment_sentiment",
      "description": "500k Reddit comments classified by Gemma 4B",
      "model_name": "google/gemma-3-4b-it",
      "max_examples": 500000,
      "batch_size": 32
    },
    {
      "domain": "math_word_problem",
      "description": "500k math word problems solved by Gemma 4B",
      "model_name": "google/gemma-3-4b-it",
      "max_examples": 500000,
      "batch_size": 32
    }
  ]

INPUT FILE RESOLUTION:
  reddit_comment_sentiment -> text_generation/reddit_comment_sentiment/reddit_comments.jsonl
  math_word_problem        -> text_generation/math_word_problem/math_word_problems.jsonl

OUTPUT DIRECTORY:
  batches/{domain}/{sanitized_model_name}/
  sanitize_model_name("google/gemma-3-4b-it") -> "google_gemma-3-4b-it"

    "domain": "math_word_problem",

    "started_at": "2026-03-04T...",
    "completed_at": null,          // -> timestamp
    "examples_generated": 0,       // -> final count
    "batches_written": 0           // -> final count
  }

DOMAIN-SPECIFIC BEHAVIOR:
  reddit_comment_sentiment:
    - Prompt: Utilities.create_evaluation_prompt(sentence) [existing]
    - Vocabulary: existing ~525 tokens (JSON response + whitespace + prompt + auxiliary)
    - Stop condition: generated token == "}"
    - Validation: all response tokens in vocabulary
    - Max generation steps: 50

  math_word_problem:
    - Prompt: master prompt (2 examples) + problem text with scaffold
    - Vocabulary: TBD after tokenizer analysis — digits, operators, variables, etc.
    - Stop condition: "Solution:" line completed (newline after final number/True/False)
    - Validation: all response tokens in math vocabulary
    - Max generation steps: TBD (scaffolds are longer than JSON responses)
    - NOTE: student only sees the scaffold, NOT the master prompt examples

MATH OUTPUT VOCABULARY (needs tokenizer analysis):
  Tokens needed:
    - Digits: 0-9 (individual + multi-digit as Gemma tokenizes them)
    - Variables: A, B, C, D, E
    - Operators: +, -, *, /, >
    - Symbols: =, \n, space, :
    - Words: Solution, True, False
    - Whitespace tokens (same as sentiment domain)
  Run tokenizer on sample scaffolds to discover exact token set.
